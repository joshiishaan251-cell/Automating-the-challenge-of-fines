# 🏛️ LegalTech Automation Pipeline

## 🎯 Глобальный контекст проекта
Мы разрабатываем отказоустойчивую локальную систему на Python для автоматизации работы юриста. 
**Специфика домена:** Цена ошибки критическая. Опечатка в сумме или неверный номер дела ведут к проигрышу в суде. 
**Главное правило (Zero Trust):** Тихие ошибки недопустимы. Если данные непонятны — система должна громко упасть (Exception) или перенести файл в `manual_review`. Запрещено удалять или перезаписывать исходные файлы юристов!

## 🛠 Технологический стек
- **Язык:** Python 3.10+ (Строгая типизация: `typing`, `dataclasses`).
- **Извлечение текста (OCR):** `pdfplumber`, `pdf2image`, `pytesseract` (Russian language pack).
- **Генерация документов:** `python-docx`, `num2words` (для сумм прописью).
- **Взаимодействие с ОС:** `pathlib` (строго для путей), `subprocess` (для запуска браузера).
- **Тестирование:** `pytest`.

## 🤖 Инструкции для ИИ-Агента (Тебя)

### 1. Правила написания кода
- Обязательно оборачивай блоки чтения/записи файлов и API-запросы в `try-except`. Логируй ошибки через модуль `logging`.
- Используй `pathlib.Path` вместо `os.path`.
- Суммы в шаблонах (DOCX) всегда форматируй с разделителями тысяч (например, `1 500 000.00`).

### 2. Рабочие процессы (Workflows)
Агент должен использовать следующие промпт-модули по требованию пользователя:
- **Анализ дел:** При ссылке на `analyze_case.md`, агент пишет OCR-скрипт для сброса текста в `case_text_dump.txt` и анализирует его, выделяя Заказчика, Перевозчика и суммы.
- **Поиск практики (Consultant/1jur):** Агент должен написать скрипт, который берет ключевые слова из `case_text_dump.txt`, формирует корректные поисковые URL-адреса и открывает их в новых вкладках строго через `CHROMIUM_PATH`.
- **Генерация претензий:** При ссылке на `claim_template.md`, агент использует `python-docx` для безопасной замены текста без поломки форматирования таблиц.

### 3. Тестирование и Ревью (Обязательный этап)
- **Тесты (`test_parser.md`):** Любая новая логика (парсинг, URL-генерация) должна покрываться `pytest` с использованием `io.BytesIO` (в памяти), чтобы не мусорить на диске. Обязательна проверка краевых случаев (опечатки, конфликты данных).
- **Скептик (`devil_advocate.md`):** В конце каждого крупного этапа агент обязан провести безжалостное ревью своего кода на предмет скрытого долга и нарушения правила "Zero Trust".
