# Модуль 1_0: Обработка постановлений и ведение базы

Этот модуль предназначен для полуавтоматической обработки сканов постановлений (PDF), проверки их на уникальность в центральном индексе и формирования локальной базы данных в формате JSON.

## Основные функции

- **Гибкий поиск папок**: Скрипт ищет все папки в корневом каталоге (например, «А1», «Б13 А40-777...»). Главное, чтобы название начиналось с буквы и цифр.
- **Выбор папки**: При запуске в **терминале** выводится нумерованный список доступных папок с указанием количества уже обработанных УИН. Пользователь вводит номер папки (1-N) и нажимает Enter.
- **Полное исключение служебных файлов**: Файлы, чьё имя начинается с любого из паттернов `ocr_skip_patterns` (по умолчанию — «Вручено», «Заявление», «заявление»), **полностью исключаются из обработки**:
  - Не открывается форма проверки
  - Не учитываются в счётчике 4-х постановлений
  - OCR для них не выполняется
  - В терминале выводится: `Пропуск (не постановление, по конфигу): …pdf`
- **Нормализация номеров авто**: После OCR-распознавания латинские буквы-двойники (A→А, B→В, E→Е, K→К, M→М, X→Х, Y→У и др.) автоматически заменяются на кириллические, так как в российских номерах нет латиницы.
- **Интеллектуальная сессия**: Скрипт запоминает уже обработанные файлы и при перезапуске пропускает их.
- **Расширенная проверка дубликатов**: Если УИН уже есть в центральной базе `uin_index.db`, скрипт покажет окно со списком всех путей и предложит **«Удалить в корзину и ПРОДОЛЖИТЬ»**. При закрытии диалога обработка останавливается.
- **Интерфейс проверки (3 поля)**: Для новых файлов открывается форма. Скрипт сам попытается найти и заполнить: Номер постановления (УИН), Дату постановления, Номер автомобиля.
- **Занесение в центральный индекс (на лету)**: После успешного заполнения формы скрипт (в зависимости от настроек) может спросить, нужно ли добавить информацию в центральный индекс.
- **Наглядный отчет**: При занесении в индекс скрипт дописывает УИН в Excel-отчет, выделяя строку красным цветом.
- **Авто-извлечение данных доставки**: Скрипт ищет в папке файл, начинающийся на «Вручено». Из его названия автоматически извлекаются:
  - `DataNumber` — дата вручения (например, `20.02.2026`)
  - `PochtaNumber` — номер РПО (например, `63097717173898`)
  - Эти данные **всегда записываются в `database.json`**, даже если все постановления уже обработаны.

> **Важно:** Скрипт обрабатывает только файлы, имеющиеся в папке **в момент запуска** (однопроходный режим). Если часть постановлений была удалена как дубликат и обработано менее 4 документов, необходимо добавить недостающие сканы в папку и **перезапустить скрипт**.

## Установка зависимостей

```bash
python -m pip install -r requirements.txt
```

## Настройка (`resolution_config.yaml`)

| Параметр | Описание |
|---|---|
| `scan_root` | Корневая папка, где лежат рабочие папки |
| `perform_ocr` | `true` / `false` — включить OCR |
| `ocr_skip_patterns` | Список префиксов имён файлов, для которых OCR **не** выполняется (напр., `Вручено`, `Заявление`) |
| `tesseract_cmd` | Путь к Tesseract OCR |
| `poppler_bin` | Путь к Poppler (нужен для конвертации PDF в изображения для OCR) |
| `libreoffice_path` | Путь к LibreOffice (нужен для генерации PDF из DOCX) |
| `template_path` | Полный путь к шаблону заявления `ShablonZayavlenie.docx` |
| `update_index_mode` | `ask` / `always` / `never` — поведение при добавлении в центральную БД |

## Порядок работы

1. Запустите **`run_1_0.bat`**.
2. В терминале введите номер папки, которую хотите обработать, и нажмите Enter.
3. Если файл требует распознавания, подождите несколько секунд (в консоли будет надпись «Выполняется OCR»).
4. Проверьте (или заполните) 3 поля в открывшемся окне и сохраните.
5. Подтвердите занесение файла в центральную базу (если в конфиге стоит `ask`).
6. После обработки всех файлов нажмите **«Да»** на вопрос о генерации заявления.
7. Готовые `.docx` и `.pdf` появятся в папке с постановлениями.

## Шаблон заявления в суд (`ShablonZayavlenie.docx`)

В шаблоне используются следующие переменные-заполнители. Скрипт поддерживает оба формата — с фигурными скобками и без:

| Переменная | Описание |
|---|---|
| `{{CASE_NUMBERS}}` | Список номеров постановлений с датами |
| `{{VEHICLE_NUMBERS}}` | Список номеров автомобилей |
| `{{POCHTA_NUMBER}}` | Трек-номер РПО (из файла «Вручено») |
| `{{DATA_NUMBER}}` | Дата вручения (из файла «Вручено») |
